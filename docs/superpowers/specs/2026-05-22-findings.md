## Theme summary

1. **IDOR is the dominant pattern.** Multiple share/read/create actions accept client IDs without `assertCan(actor, 'animal.read')` + soft-delete filter. ~7 instances.
2. **JWT is too trusting.** Role + active flag are taken from the token, never re-checked against DB. Drives ~5 findings.
3. **The Trash gap is real and structural.** Animal has no delete; Document has no restore; the UI lies about restoring from logs; soft-deleted parents leak via file proxy + reports + search.
4. **Deploy readiness is a build-blocker today.** Edge middleware will crash on Vercel because it imports Prisma transitively. Headers, env validation, region, pooling, secrets — all need addressing in one pass.
5. **Drive integration is largely correct,** but scope is too broad, parent check fails open, and one path lets users hijack each other's uploads.
6. **Tests cover less than the codebase suggests.** 20 scripts not in CI; only 3 of 9 features have unit tests; no negative RBAC paths.

## Fix order (proposed)

1. **VRC-1** (edge middleware) — fix first or no deploy.
2. **AUTH-1 + AUTH-2 + AUTH-3** (JWT re-check + rate limit + rotate secret) — auth bedrock.
3. **API-1 + RBAC-1 + STO-1 + STO-3** (file IDOR, staging session, dup-asset, fail-open metadata) — close the file-system blast radius.
4. **ACT-1 + ACT-2 + ACT-3 + RBAC-2 + RBAC-5** (share/read IDOR + lifecycle file ownership + listAllDocuments) — close server-action blast radius.
5. **Build Trash feature** (SD-1, fixes UI-1) — user's flagged item; carries fixes for SD-2/SD-4/SD-5/SD-6 by way of consistent soft-delete filters.
6. **VRC-2..5** (headers, env guard, pooling, secret rotation).
7. **Remaining Critical + High** in order.
8. **Medium** in batches by file.
9. **Low/Info** quick batch at the end.
10. **Run all e2e + write new probes** (Trash, RBAC, soft-delete consistency).
11. **Final deploy-readiness gate.**
