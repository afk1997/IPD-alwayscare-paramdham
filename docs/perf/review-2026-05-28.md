## Fixed in PR #23 (`0d8a999`)

### 1. `ActivityTimeline` takes explicit `animalId` prop

**Was:** `ownAnimalId = activities[0]?.animalId ?? initial[0]?.animalId` — when a patient had zero activities, the inferred owner was `undefined` and QuickAdd dispatches never spliced into the timeline.

**User-felt symptom:** Admit a new patient → log first activity via QuickAdd → activity invisible until page reload.

**Fix:** `AnimalDetail.tsx` passes `animalId={animal.id}` explicitly. Timeline's feed-subscriber effect uses it directly. Deps tightened from `[lastEvent, activities, initial]` to `[lastEvent, animalId]`.

**Commit:** `0c307e1`

### 2. All three virtualizers switched to `useWindowVirtualizer`

**Was:** `<div ref={parentRef} className="max-h-[78vh] overflow-y-auto md:max-h-[80vh]">` inside `AppShell`'s outer `flex-1 overflow-auto` scroller → nested scroll containers. iOS Safari touch-handoff between scrollers is notoriously janky.

**User-felt symptom:** On mobile, the patient page scrolls until the timeline reaches top, then the gesture has to "enter" the inner region. Reverse scroll has to bottom out the inner before the page resumes.

**Fix:** `useWindowVirtualizer` from `@tanstack/react-virtual`. Drops the inner `max-h` + `overflow-y-auto`. Items position absolutely relative to the list via `vi.start - virtualizer.options.scrollMargin`. Single page scroll restored; virtualization perf preserved.

**Files:** `ActivityTimeline.tsx`, `PatientListVirtual.tsx`, `TodayTimelineList.tsx`

**Commit:** `4d40828`

### 3. Error toast on Undo restore failure

**Was:**
```tsx
onClick: async () => {
  const r = await restoreActivityAction(id);
  if (r.ok && r.activity) onRestored(r.activity);
}
```
No `else` branch. If `restoreActivityAction` failed (RBAC denial, network blip), nothing visible happened.

**Clinical risk:** Doctor accidentally deletes a SURGERY → clicks Undo → silently fails → row appears lost (still in Trash, but only admins can recover).

**Fix:** `showToast({ message: r.error ?? 'Could not restore — check Trash page' })` on the `!r.ok` branch.

**File:** `src/features/activities/components/ActivitySheet.tsx`

**Commit:** `856eceb`

### 4. `DocumentList` renders disabled state for null `fileUrl`

**Was:** `<Link href={d.fileUrl ?? '#'} target="_blank">` — for Document rows with no attached file (legitimate state), clicked open a new tab navigating to `#`. Silent broken UX.

**Fix:** Conditional render. When `fileUrl` is null, render a dimmed `<div>` with `opacity-60` and a `text-critical` "File unavailable" subtitle.

**File:** `src/features/documents/components/DocumentList.tsx`

**Commit:** `45b692a`

### 5. `createActivityAction` / `updateActivityAction` return instead of throw

**Was:** Both actions ended their catch blocks with `throw e`. Every other action in the codebase returns `{ ok: false, error: '...' }`.

**Failure mode:** In `ActivitySheet.save()`, the action runs inside `useTransition` with no try/catch. A thrown error → uncaught promise rejection → optimistic state never reverted.

**Fix:** `console.error(...); return { ok: false, error: 'Could not …' };` — pattern now matches `deleteActivityAction`, `restoreActivityAction`, `duplicateActivityAction`.

**File:** `src/features/activities/actions.ts`

**Commit:** `f4ffa85`

### 6. Seed creates an E2E fixture animal + media when `STORAGE_DRIVER=local`

**Was:** `signed-media.spec.ts` and `optimistic-mutations.spec.ts` both `test.skip` when DB has no media. `prisma/seed.ts` only seeded 6 users → tests passed in CI **only by skipping silently**.

**Fix:** Seed extension creates one fixture animal + one READY `MediaAsset` (1×1 JPEG written to `LOCAL_UPLOAD_DIR`) + one `AnimalMedia` link + one Activity. Gated on `STORAGE_DRIVER=local`. Idempotent (checks for existing `E2E-Fixture` animal).

**File:** `prisma/seed.ts`

**Commit:** `bf82477`

### 7. `next.config.ts` declares `images.localPatterns`

**Was:** Next 15.5 emits a runtime warning: "query string not configured in `images.localPatterns`; this will be REQUIRED starting in Next.js 16."

**Failure mode:** On `pnpm up next@16`, every `<Image>` referencing a signed URL would throw — full visual regression.

**Fix:**
```ts
localPatterns: [
  { pathname: '/api/files/**', search: '' },
  { pathname: '/api/files/**', search: '?v=orig&sig=*' },
],
```

**File:** `next.config.ts`

**Commit:** `e53f9c3`

## Backlog — MEDIUM severity

Not blocking anything; deferred from the cleanup PR. Listed in order of user-felt impact.

### M1. `useEffect(() => setX(initial), [initial])` clobbers optimistic state

**Files:** `ActivityTimeline.tsx:62`, `TodayTimelineList.tsx:175-177`, `AnimalDetailsTab.tsx:27`

The naive re-sync runs on every parent re-render that creates a new array reference. In-flight optimistic edits get wiped.

**Why deferred:** Hard to repro in practice. The Next.js RSC data layer typically delivers an `initial` that already includes the optimistic mutation. But fragile.

**Fix:** Either drop the sync effect entirely (mount-only state init is correct for a client component that owns its mutations), or guard with a shallow content check:
```ts
useEffect(() => {
  setActivities((prev) =>
    prev.length === initial.length && prev[0]?.id === initial[0]?.id ? prev : initial
  );
}, [initial]);
```

Note: `CagesPanel` and `DocumentsPanel` already don't have this pattern. Removing it from the three that do gives consistency.

### M2. `TodayTimelineList` renders blank animal info for QuickAdd-dispatched rows

**File:** `TodayTimelineList.tsx:79-95, 200-203`

`activityToFeedItem(a)` without a `prev` row → `animalName: ''`, `animalSpecies: ''`, `animalThumbnailUrl: null`. The today timeline shows a row with empty patient identity until the next navigation refreshes.

**Why deferred:** Mostly hidden by the `router.push` to the patient page that follows QuickAdd — the blank row exists for milliseconds. But if QuickAdd fires from a context where the user stays on `/`, the blank row sticks.

**Fix:** Either (a) extend `SerializedActivity` with the patient triple (`animalName`, `animalSpecies`, `animalThumbnailUrl`) — server already has it via `Activity.animal` join, just include it in `serializeActivity`; or (b) keep a `Map<animalId, AnimalLite>` in `TodayTimelineList` seeded from `initial` and look up on dispatch.

### M3. `MediaGrid columns=4` `sizes` prop doesn't match actual responsive layout

**File:** `MediaGrid.tsx:23`

On mobile the grid is `grid-cols-3` regardless of the `columns` prop. For `columns={4}` callers (e.g., `AnimalDetail` admission media), `sizes="(max-width: 768px) 25vw, 150px"` is wrong on mobile (actual = 33vw) and undersized at 4-col desktop layouts. Sources slightly too small → blur on hi-DPI.

**Fix:**
```ts
const sizesValue = columns === 4
  ? '(max-width: 768px) 33vw, 25vw'
  : '(max-width: 768px) 33vw, 33vw';
```

### M4. Optimistic save is reentrant — double-click fires twice

**File:** `ActivitySheet.tsx` (`save`, `del`)

Save button `disabled={pending}` but `pending` only flips after `startTransition` schedules work. Double-click between optimistic apply and `pending` toggling → two server roundtrips + two "Updated" toasts.

**Fix:**
```ts
const isSubmittingRef = useRef(false);
const save = () => {
  if (isSubmittingRef.current) return;
  isSubmittingRef.current = true;
  // ... existing logic
  start(async () => {
    try { /* server call */ } finally { isSubmittingRef.current = false; }
  });
};
```
Same pattern in `del()`.

### M5. `AnimalHero` `sizes` off-by-one at exactly 768px

**File:** `AnimalHero.tsx:69, 80`

`sizes="(max-width: 768px) 68px, 78px"` with Tailwind `md:` (`min-width: 768px`). At exactly 768px both branches match; browser picks first → 68px source, but CSS renders 78px. 1px viewport edge case.

**Fix:** Change to `(max-width: 767px) 68px, 78px`.

### M6. `MediaUploader sizes="200px"` ignores grid breakpoint

**File:** `MediaUploader.tsx:143`

Parent grid is `grid-cols-3 md:grid-cols-4`. Hardcoded 200px is over-sourced on mobile, slightly under at hi-DPI desktop.

**Fix:** `sizes="(max-width: 768px) 33vw, 25vw"`.

## Backlog — LOW severity

### L1. Dead `dispatchActivityRemoved` export

**File:** `src/lib/hooks/useActivityFeed.ts:50`

The function exists and is tested but never called outside tests. `ActivitySheet.del()` calls the local `onDeleted(id)` callback only. So deletes propagate to the current component but NOT cross-component via the bus.

**Why deferred:** No current page mounts both `ActivityTimeline` and `TodayTimelineList`, so the asymmetry isn't user-visible. But it's a footgun.

**Fix:** Either delete `dispatchActivityRemoved` (and its test), or wire it from `ActivitySheet.del`:
```tsx
onDeleted(id);
dispatchActivityRemoved(id);
onClose();
```

### L2. `serializeActivity(row: any)` lacks compile-time guard

**File:** `src/features/activities/serialized.ts:23, 38`

The helper accepts `row: any` with biome-ignore. Callers in `actions.ts` happen to pass joined rows that include `media.asset.kind`, but if a future caller forgets the `include`, `m.asset.kind` throws at runtime.

**Fix:** Type the input properly:
```ts
type ActivityWithJoins = Activity & {
  media: (ActivityMedia & { asset: MediaAsset })[];
};
function serializeActivity(row: ActivityWithJoins, signUrl: (assetId: string) => string): SerializedActivity {
```

Adds compile-time enforcement that callers actually `include` what they need.

### L3. `signMediaUrl` doesn't validate `assetId`

**File:** `src/lib/media-sign.ts:18-28`

`signMediaUrl('')` would happily sign `|orig`. No reachable caller passes empty today.

**Fix:** Defensive check:
```ts
if (!assetId || assetId.length < 8) throw new Error('signMediaUrl: invalid assetId');
```

### L4. `AUTH_SECRET` length not validated

**File:** `src/lib/media-sign.ts:21`

A 1-char secret silently passes HMAC and produces a near-broken signature.

**Fix:** Fail-fast at sign time:
```ts
if (secret.length < 32) throw new Error('AUTH_SECRET too short (need ≥32 chars)');
```

### L5. Middleware matcher uses prefix lookahead

**File:** `src/middleware.ts:29`

`(?!api/files|...)` matches as a string prefix. A future `/api/files-export` route would silently inherit the NextAuth bypass.

**Fix:** Anchor with segment terminator:
```ts
matcher: ['/((?!api/auth(?:/|$)|api/files(?:/|$)|_next/static|_next/image|favicon\\.ico).*)'],
```

### L6. `Lightbox` iframe (DOC type) has no `sandbox` attribute

**File:** `src/components/media/Lightbox.tsx`

CSP at the route layer already restricts PDF behavior, but defence-in-depth would add `sandbox="allow-same-origin"`.

### L7. `cache-control: immutable` on signed URLs + 1-year `minimumCacheTTL`

**Files:** `src/app/api/files/[id]/route.ts` (signed branch), `next.config.ts:images.minimumCacheTTL`

`immutable` tells the browser to never revalidate within `max-age`. Combined with Next image cache held for 1 year, rotating `AUTH_SECRET` doesn't invalidate already-served bytes — old URLs keep working from cache up to their TTL.

**Why deferred:** Spec explicitly accepted this trade. Documented for the runbook: secret rotation requires Vercel cache purge for full invalidation.

### L8. `getAnimal` spreads internal Prisma fields into RSC boundary

**File:** `src/features/animals/queries.ts:119-122`

`{ ...m, url: signMediaUrl(m.asset.id) }` spreads `storageKey: 'gdrive:1abc…'` and other internal fields across the wire. Not a credential, just payload bloat per patient detail page.

**Fix:** Map to a thin shape:
```ts
media: animal.media.map((m) => ({
  id: m.id,
  label: m.label,
  url: signMediaUrl(m.asset.id),
  asset: { id: m.asset.id, filename: m.asset.filename, kind: m.asset.kind },
})),
```

### L9. `byUser` included on every activity write but never read

**File:** `src/features/activities/service.ts:61, 119, 162, 200, 249`

Every mutation includes `byUser: { select: { id, name } }`. But `serializeActivity` reads `row.byName` (the denormalized column) instead. Dead include = one wasted DB join per write.

**Fix:** Drop the include, OR switch `serializeActivity` to read `row.byUser.name` as the canonical source.

### L10. CI grep gate regex only catches backtick-template literal form

**File:** `scripts/check-no-raw-file-urls.sh:7`

Pattern: `` `/api/files/${ ``. A future commit could do `"/api/files/" + assetId` (double-quoted concat) or `'/api/files/' + assetId` (single-quoted concat) and slip through.

**Fix:** Widen to OR-grep across all three quote chars:
```bash
grep -rnE --include='*.ts' --include='*.tsx' \
  "['\"\`]/api/files/" src/ --exclude=media-sign.ts
```

### L11. Tab unmount drops feed subscription

**File:** `AnimalDetailTabs.tsx`

Tab switching does `{tab === 'feed' && feed}` — `ActivityTimeline` unmounts when not on the Feed tab. New `useActivityFeed` events that fire while on a different tab are lost; switching back remounts from server-provided `initial` (which may not yet reflect the dispatched event).

**Why deferred:** Pre-existing limitation introduced by Phase 2's feed store, not by virtualization. Not a regression vs the previous `router.refresh` model (which also lost state on tab switch). Worth a future fix that keeps the timeline mounted with `hidden` toggle, or threads the feed subscription up to a stable parent.

### L12. Inconsistent `overscan` (5 vs 6) across virtualizers — RESOLVED in cleanup PR

`PatientListVirtual` originally had `overscan: 6`. Cleanup PR aligned all three to `5` (consistent with TanStack default). Listed here as a closed item.

## Verified safe — no action needed

Multi-agent confirmed; do not re-litigate.

### Security & auth boundary

- **HMAC correctness:** `timingSafeEqual` correctly preceded by buffer length-equality check. Multi-byte UTF-8 tricks rejected at the buffer-length stage before reaching the timing comparison.
- **React `cache()` request-isolation:** Per React's contract, the cache scope is per-request-graph. Server Actions get their own scope. No cross-request user data leak possible.
- **`signMediaUrl` reachability:** Only called server-side (queries, server actions, finalize route). No client→server endpoint mints URLs without auth.
- **`/api/files/finalize` ordering:** RBAC check fires before `signMediaUrl(asset.id)`. No "403 + usable URL" race.
- **`/api/files/initiate` and `/api/files/finalize` auth:** Both routes call `getCurrentUser` internally. Middleware bypass for `/api/files/*` does not create an unauthenticated path.
- **`URLSearchParams` parameter pollution:** Duplicate `sig` or `v` keys → `get('sig')` always returns the first. Sign and verify sides are deterministic and consistent. No exploit.
- **Local storage path traversal:** `LocalDiskStorage.fromKey()` → `absolute()` → root check (`abs !== root && !abs.startsWith(root + sep)`). Production refuses `local` driver entirely.
- **ETag format `"<id>-<variant>"`:** Doesn't leak more than the URL itself; not an enumeration channel.

### Build / dependencies

- **No Prisma schema change** in the rebuild range — `git diff ac3c8ed..main -- prisma/schema.prisma` is empty.
- **Only one new dependency:** `@tanstack/react-virtual@3.13.26` + transitive `@tanstack/virtual-core@3.16.0`. No drive-by adds.
- **No new `any` introduced** beyond `serializeActivity(row: any)` (covered as L2).
- **No `TODO` / `FIXME` / `XXX`** added in rebuild commits.

### State management

- **Server Actions and React `cache()`:** Each Server Action is its own request graph; the cache works as expected within each.
- **`useSyncExternalStore` + module-level state:** SSR safe — `getServerSnapshot()` returns `null`, so the bus only matters client-side. No hydration mismatch.
- **`useActivityFeed` listener cleanup:** Components remove themselves via `useEffect` cleanup on unmount. No leak.

## Phase 5 / 6 deferral reasoning

After Phase 4 shipped, the spec called for Phase 5 (split `AppShell` server/client + demote presentational client components) and optional Phase 6 (PWA).

**Decision: deferred indefinitely.** Reasoning:

| Metric | After Phase 1-4 (current) | Estimated after Phase 5 | Marginal swing |
|---|---|---|---|
| TTI on `/patients` (mobile) | ~1.8 s | ~1.7 s | ~50-100 ms |
| Hydration JS for shell | ~25 KB | ~10 KB | ~15 KB |
| First-paint feel | smooth | smooth | imperceptible |

Compare to wins already shipped:
- Phase 1: 2,175 ms → 62 ms per image · 100× mobile-data drop. **Transformative.**
- Phase 2: sheet save 1–3 s flicker → instant. **Huge qualitative.**
- Phase 3: 300-row list jank → smooth. **Meaningful for long-stay patients.**
- Phase 4: ~50–150 ms saved per page render via DB dedup. **Moderate.**
- Phase 5 estimate: ~50–100 ms TTI, mostly imperceptible. **Small.**

The perf budget set in the spec was met after Phase 1-4 (FCP < 1.5s ✓, signed `/api/files` < 50ms warm ✓, `/patients` LCP < 2.5s ✓). Phase 5's incremental gain doesn't justify the risk of refactoring `AppShell` (which wraps every page).

Phase 6 (PWA) is unrelated to the perf budget — it's about offline capability and network resilience. Deferred until a real product need emerges.

**Both phases stay doable later;** nothing about Phase 1-4 makes them harder. The umbrella spec at `docs/superpowers/specs/2026-05-28-app-performance-design.md` still has full designs for both.
