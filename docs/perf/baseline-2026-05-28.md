**Per-stage time (subtracting from previous mark):**

| Stage                                                            | Median (ms) | % of total |
|------------------------------------------------------------------|------------:|-----------:|
| `auth` alone (`getCurrentUser()` â€” DB user lookup)               | **134**     | 6.2% |
| `getMediaForRead` (1 asset query + up to 3 parallel link checks) | **113**     | 5.2% |

**Drive is the entire performance story.** Phase 1's signed-URL path drops
both `auth` and `getMediaForRead` chains (no cookie auth, single asset
lookup) and Phase 1's `getStreamOnly` drops the Drive metadata RTT â€” leaving
just one Drive stream call, edge-cached after the first hit.

### Server-Timing â€” production reality

The Phase 0 `server-timing` header is **stripped by Vercel's edge layer** in
production responses for this project â€” three deployments + an empty-commit
rebuild all consistently returned no `server-timing` header on `/api/files`
responses despite the route handler clearly setting it. The instrumentation
works locally; Vercel either filters this header by default on Hobby tier or
something in the response transformation drops it. **Captured in this doc
only;** not a Phase 1 blocker since per-image network time (62 ms cold, 0 ms
warm) is a direct, externally observable substitute.

### What this means for the user-felt experience

- The Vercel "Function Invocations > Edge Requests" inversion will **flip**
  within the next 30-day cycle as image requests start absorbing into the
  edge cache instead of repeatedly invoking the function.
- Mobile data per page view drops from **~50 MB** (17 cards Ã— ~3 MB each
  full-size JPEG) to **~500 KB** (17 Ã— ~30 KB AVIF) â€” **100Ã— reduction**.
- Re-visits to the same page no longer pay any image cost.
- The Drive `files.get` metadata RTT (~half of Drive's per-image latency)
  is dropped on the signed path via `getStreamOnly`.

## How these numbers were captured

1. `git checkout perf-rebuild-plan` (commit `4163850`, includes Phase 0 instrumentation).
2. `pnpm build && pnpm exec dotenv -e .env.local -- env AUTH_URL=http://localhost:3000 pnpm exec next start --port 3000`.
3. `tmp/capture-baseline.mjs` (Playwright Pixel 7 emulation) signs in as the
   test user, navigates to `/patients`, captures every `/api/files/*` response
   header, prints them as JSON.
4. `npx -y lighthouse@12 http://localhost:3000/<path> --form-factor=mobile
   --throttling-method=simulate --screen-emulation.mobile=true
   --extra-headers='{"Cookie":"<session>"}'` for each of the three pages above.
5. Vercel dashboard snapshot read directly from the project's Usage tile.

`tmp/capture-baseline.mjs`, `tmp/lh-*.json`, and `tmp/cookie.txt` are scratch
artifacts in `tmp/` which is git-ignored â€” they are NOT committed.
